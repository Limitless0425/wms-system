# WMS 仓储管理系统 — 需求分析文档

## 1. 引言

### 1.1 项目背景
本项目为汽车零部件仓储管理系统（WMS），面向汽车主机厂及其零部件供应商的入库、出库、转包、库存管理等核心业务场景。系统参考一汽大众 WMS 上下游汽车物流供应链流程设计，覆盖从供应商发货到仓库收货入库、库存管理、转包作业、出库发货的全链路。

### 1.2 项目目标
构建一套完整的仓储管理信息系统，实现：
- 零部件入库全流程管理（入库单 → 看板 → 扫码入库）
- 零部件出库管理（带单出库 / 不带单出库）
- 转包作业管理（普通转包 + 转包结余）
- 基础资料管理（供应商、客户、零件、仓库、库位、器具）
- 库存追溯与监控
- AI 辅助仓库管理
- 移动端（手持终端）扫码作业

### 1.3 技术选型
| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Element Plus + Vite | SPA 单页应用 |
| 后端 | Spring Boot 3.2.5 | RESTful API |
| 安全 | Spring Security + JWT + BCrypt | 认证与授权 |
| 持久层 | Spring Data JPA + H2 | 文件型数据库 |
| 构建工具 | Maven (后端) + npm (前端) | 依赖与打包 |

### 1.4 用户角色
| 角色 | 权限 | 说明 |
|------|------|------|
| ADMIN | 全部权限 | 系统管理员，默认账号 admin/admin123 |
| OPERATOR | 仓库作业权限 | 仓库操作员 |
| AI_ADMIN | 全部权限 | AI 管理员，默认账号 ai_admin/ai123456 |

## 2. 功能需求

### 2.1 基础信息管理

#### 2.1.1 供应商管理
- **功能描述**：维护零部件供应商基础信息
- **数据字段**：供应商编号、供应商名称、联系人、联系电话、地址、状态（启用/停用）、创建时间
- **操作**：增删改查、批量导入（文本 + Excel）
- **搜索**：按供应商名称、供应商编号、状态搜索
- **业务规则**：供应商编号唯一，停用的供应商不能用于新建入库/出库/转包单

#### 2.1.2 客户管理
- **功能描述**：维护下游客户信息
- **数据字段**：客户编号、客户名称、联系人、联系电话、地址、状态、创建时间
- **操作**：增删改查、批量导入
- **搜索**：按客户名称、客户编号、状态搜索

#### 2.1.3 零件管理
- **功能描述**：维护汽车零部件基础信息
- **数据字段**：零件编号、零件名称、规格、单位、供应商、安全库存（低储）、最大库存（高储）、客户条码、原包装容量、转包容量、转包器具类型
- **操作**：增删改查、批量导入（文本 + Excel）
- **搜索**：按零件编号、零件名称、供应商、零件类型搜索
- **业务规则**：
  - 零件编号唯一
  - 原包装容量和转包容量必须大于 0
  - 转包容量不能大于原包装容量
  - 转包器具类型不能为空
  - 库存低于低储阈值或高于高储阈值时产生告警

#### 2.1.4 仓库管理
- **功能描述**：维护仓库信息
- **数据字段**：仓库编号、仓库名称、仓库总容量、地址、状态、创建时间
- **操作**：增删改查、批量导入
- **展示**：总容量、库位容量合计、剩余容量（总容量 - 实际库存）、超容警告
- **业务规则**：新增库位时若超出仓库总容量则警告

#### 2.1.5 库位管理
- **功能描述**：维护仓库内库位信息
- **数据字段**：库位编号、库位名称、所属仓库、容量、状态
- **操作**：增删改查、批量导入
- **筛选**：按仓库筛选库位
- **业务规则**：
  - 库位必须属于一个仓库
  - 库位容量可修改
  - 新建入库单时若库位容量不足需警告

#### 2.1.6 器具管理
- **功能描述**：维护仓储器具信息（普通器具 + 转包器具）
- **数据字段**：器具代码、器具名称、器具类型（普通/转包）、包装容量、包装单位、状态
- **操作**：增删改查、批量导入
- **搜索**：按器具代码、器具名称、器具类型搜索
- **业务规则**：
  - 器具代码唯一
  - 转包器具用于转包作业中推荐目标器具
  - 系统初始化预置 RPK010/RPK020/RPK030/RPK050/RPK100 等转包器具

---

### 2.2 入库管理

#### 2.2.1 入库单管理
- **功能描述**：创建和管理入库单
- **入库单数据字段**：入库单号（自动生成 RK-时间戳）、订单号、供应商、状态、创建时间、操作人
- **明细数据字段**：零件编号、零件名称、数量、仓库、库位、器具、包装容量
- **操作**：新建、修改、删除、作废、批量添加零件、查看关联看板
- **搜索**：按供应商（下拉选择）、状态、入库单号、订单号筛选
- **批量添加零件**：
  - 选择供应商后显示该供应商所有零件列表，支持逐行勾选并填写数量、器具、包装容量、仓库、库位
  - 支持文本识别：粘贴需求文本后自动解析零件信息
  - 支持 Excel 导入
  - 每个明细独立选择仓库和库位
- **业务规则**：
  - 新建入库单后自动生成看板（每条明细一个看板）
  - 已生成看板的入库单可修改，修改后关联看板自动更新；新增明细自动生成新看板
  - 可作废未入库的入库单
  - 部分入库后，剩余未入库零件自动生成新看板
  - 删除入库单同时删除关联看板和库存信息

#### 2.2.2 看板管理
- **功能描述**：管理所有入库看板
- **看板数据字段**：看板号（自动生成 KB-时间戳-随机码）、入库单号、供应商、零件编号、零件名称、数量、仓库、库位、状态、是否封存、二维码
- **操作**：查看、打印看板（含二维码）、批量打印和导出
- **搜索**：按状态（未入库/已入库/待扫描/封存/转包入库/已作废）、入库单号、仓库、看板号、供应商（下拉选择）、零件号（下拉选择）搜索
- **排序**：最新的看板排在列表上方
- **业务规则**：
  - 看板状态流转：未入库 → 已入库（扫码入库后）
  - 可封存/解封看板
  - 转包看板状态：未入库/已入库/转包入库/已转包

#### 2.2.3 扫码入库
- **功能描述**：通过扫码（或输入看板号）完成入库
- **操作**：
  - 输入看板号或使用摄像头扫描看板二维码
  - 支持批量入库：勾选多条看板后弹出弹窗，逐行可选择入库数量，一键执行
  - 入库后生成库存记录和库存流水
- **界面**：分为"未入库看板"列表和"最近入库记录"列表，最新记录排在列表上方
- **业务规则**：
  - 仅未入库且未封存的看板可扫码入库
  - 支持部分入库
  - 入库后更新库存数量和看板状态

---

### 2.3 出库管理

#### 2.3.1 出库单管理
- **功能描述**：创建和管理出库单
- **出库单数据字段**：出库单号（自动生成 CK-时间戳）、供应商、客户、出库类型（出库/退库/调账出库/调账退库）、状态、创建时间
- **明细数据字段**：零件编号、零件名称、数量、来源库存看板号
- **操作**：新建、修改、删除、作废、批量添加零件、查看关联看板
- **搜索**：按供应商（下拉选择）、出库类型、状态搜索
- **批量添加零件**：支持文本识别和 Excel 导入
- **业务规则**：
  - 新建出库单后自动生成出库看板
  - 出库类型可区分正常出库和退库
  - 保存前校验库存是否充足
  - 出库看板扫码后按 FIFO 扣减库存

#### 2.3.2 带单出库
- **功能描述**：按出库单执行带单出库
- **操作**：
  - 扫描出库看板号或使用摄像头扫码
  - 显示出库看板详情（零件、数量、仓库、库位等）
  - 支持批量出库：勾选多条看板后弹出弹窗，逐行选择出库数量
- **逻辑**：FIFO 扣减，按来源库存看板入库时间最早优先扣减

#### 2.3.3 不带单出库
- **功能描述**：直接扫描库存看板出库，无需出库单
- **操作**：
  - 扫描库存看板号或使用摄像头扫码
  - 显示库存看板详情
  - 选择出库数量后直接出库
  - 支持批量出库
- **逻辑**：直接从库存扣减，生成库存流水

---

### 2.4 转包管理

#### 2.4.1 转包单管理
- **功能描述**：创建和管理转包单
- **转包单数据字段**：转包单号（自动生成 ZB-时间戳）、供应商、来源看板号、来源业务类型、状态、备注、创建时间
- **转包明细**：零件编号、零件名称、原包装容量、转包数量、转包器具类型
- **操作**：新建、删除、打印转包单、打印看板
- **搜索**：按转包单号、供应商、转包状态搜索
- **业务规则**：
  - 新建转包单时校验库存是否充足
  - 转包器具推荐优先匹配零件维护的转包容量
  - 转包单不直接执行转包，通过扫描转包看板完成

#### 2.4.2 转包作业（手持转包）
- **功能描述**：扫描看板执行转包
- **操作**：
  - 扫描看板号或使用摄像头扫码
  - 支持扫描：未入库看板、已入库看板、未出库看板、转包看板
  - 选择实际转包数量
  - 选择目标仓库、库位、器具
  - 未转包记录 + 最近转包记录列表
  - 支持批量转包
- **转包逻辑**：
  - **扫未入库看板**：未全部转包 → 原看板数量减少，状态仍为未入库，生成转包单和转包看板，转包看板扫描后状态为"转包入库"；全部转包 → 原看板状态变为"已转包"
  - **扫已入库看板**：生成转包单和看板，转包看板扫描后先扣减原库存再增加目标库存，原看板标记为"已转包"，转包看板标记为"转包入库"
  - **扫未出库看板**：与未入库同理，方向为"转包出库"
  - **转包看板可删除**（扫描前），删除不影响库存
  - **库存流水完整追溯**

#### 2.4.3 转包结余
- **功能描述**：查看转包结余情况
- **数据字段**：转包单号、零件编号、计划数量、完成数量、转包结余
- **搜索**：按转包单号、零件编号搜索
- **业务规则**：转包完成后，未完成的数量记录为转包结余

## 3. 数据模型

### 3.1 核心实体及其关系

```
User ──── Role (多对一)
Supplier (独立)
Customer (独立)
Part ──── Supplier (多对一)
Part ──── RepackRule (一对一/多对一，转包规则)
Warehouse ──── Location (一对多)
Container (独立)

InboundOrder ──── InboundOrderItem (一对多)
InboundOrderItem ──── Part (多对一)
InboundOrder ──→ Kanban (一对多，自动生成)

OutboundOrder ──── OutboundOrderItem (一对多)
OutboundOrderItem ──── Part (多对一)
OutboundOrder ──→ OutboundKanban (一对多，自动生成)

RepackOrder ──── RepackOrderItem (一对多)
RepackOrder ──→ RepackKanban (一对多)
RepackOrderItem ──── Part (多对一)

Kanban ──→ InboundOrder (来源入库单)
InventoryRecord ──→ Part / Warehouse / Location / Kanban (关联追溯)

RepackRecord (转包作业记录)
```

|------|----------|------|

| 入库看板 | KB-时间戳-随机码 | KB1782311114980-9C5B43 |
| 出库单 | CK-时间戳 | CK-1782377323211 |
| 出库看板 | OKB-时间戳-随机码 | OKB1782311114980-9C5B43 |
| 转包单 | ZB-时间戳 | ZB-1782377323211 |
| 转包看板 | RKB-时间戳-随机码 | RKB1782368704819302 |

## 4. 业务流程

### 4.1 入库流程
```
创建入库单 → 选择供应商 → 批量添加零件(选择仓库/库位/器具/数量)
→ 保存入库单 → 自动生成看板(每条明细一个) → 打印看板
→ 扫码入库(支持部分入库/批量入库) → 更新库存 → 记录库存流水
```

### 4.2 出库流程
```
方式一(带单出库)：创建出库单 → 批量添加零件 → 保存 → 自动生成出库看板
→ 扫码出库 → FIFO扣减库存 → 记录流水

方式二(不带单出库)：扫码库存看板 → 选择出库数量 → 直接出库 → 记录流水
```

### 4.3 转包流程
```
创建转包单(或扫描看板自动创建) → 生成转包看板
→ 扫描转包看板 → 选择数量/仓库/库位/器具
→ 扣减原库存/原看板 → 增加目标库存/生成新看板 → 记录流水
→ 查看转包结余
```

## 5. 非功能性需求

### 5.1 性能要求
- 页面首次加载时间 < 3 秒
- 列表查询响应时间 < 1 秒
- 支持并发操作

### 5.2 安全要求
- 用户密码 BCrypt 加密存储
- JWT Token 认证，登录过期自动跳转
- 基于角色的权限控制（RBAC）
- 前端路由守卫 + 后端接口权限校验

### 5.3 可用性
- 搜索框均有 placeholder 提示词
- 操作按钮有明确的确认/取消提示
- 错误操作有明确的错误提示
- 支持桌面端和移动端两套界面

### 5.4 可维护性
- 前后端分离，代码结构清晰
- 后端分层：Controller → Service → Repository
- 前端组件化：公用组件复用（OrderItemBatchPicker）

## 6. 接口概览

### 6.1 认证接口
- `POST /api/auth/login` — 用户登录
- `GET /api/menus` — 获取用户菜单

### 6.2 基础信息接口
- `/api/suppliers` — 供应商 CRUD + 批量导入
- `/api/customers` — 客户 CRUD + 批量导入
- `/api/parts` — 零件 CRUD + 批量导入
- `/api/warehouses` — 仓库 CRUD + 批量导入
- `/api/locations` — 库位 CRUD + 批量导入（按仓库筛选）
- `/api/containers` — 器具 CRUD + 批量导入

### 6.3 入库接口
- `/api/inbound/orders` — 入库单列表/新建/修改/删除/作废
- `/api/inbound/kanbans` — 看板列表/打印/导出
