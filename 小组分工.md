## 成员2：后端入库模块 + 库存模块 + 需求分析文档

**入库单管理**
- InboundOrder、InboundOrderItem 实体与 Repository
- 入库单新建接口（按供应商筛选零件 → 逐行选数量/仓库/库位/器具 → 保存）
- 入库单修改接口（自动更新关联看板）
- 入库单作废、删除接口（级联删除看板和库存）
- 入库单搜索接口（按供应商、状态、入库单号、订单号筛选）

**看板管理**
- Kanban 实体与 Repository
- 入库单保存后自动为每条明细生成看板（编号：KB-时间戳-随机码）
- 看板列表查询（按状态/入库单号/仓库/看板号/供应商/零件号搜索）
- 看板二维码生成、打印、批量导出接口
- 看板封存/解封接口

**扫码入库**
- 扫码入库接口（输入看板号或摄像头扫码 → 显示详情 → 确认入库）
- 批量入库（勾选多条看板 → 逐行选数量 → 一键入库）
- 部分入库（总数量 10，可先入 3，剩余 7）
- 入库后更新库存数量、看板状态、记录库存流水

**库存模块**
- 库存总览：按零件汇总已入库数量
- 库存追溯：按零件/时间/操作类型追溯库存变动历史
- 库存报表统计
- 前端库存总览 `StockSummary.vue`、库存追溯 `InventoryTrace.vue`、库存报表 `InventoryReport.vue`

**需求分析文档**
- 功能需求章节（入库、库存）
- 业务流程（入库流程、扫码入库流程）
- 数据模型说明

## 成员3：后端出库模块 + 转包模块 + 测试文档

**出库单管理**
- OutboundOrder、OutboundOrderItem、OutboundKanban 实体与 Repository
- 出库单新建接口（选供应商和出库类型 → 批量添加零件 → 校验库存充足 → 保存）
- 保存后自动生成出库看板（编号：OKB-时间戳-随机码）
- 出库单修改、作废、删除接口

**带单出库**
- 扫码出库接口（扫出库看板号 → 按来源库存看板 FIFO 扣减）
- FIFO 扣减（按入库时间最早优先扣减，一张库存看板不够则跨看板连续扣）
- 批量出库（勾选多条看板 → 逐行选数量 → 一键出库）
- 库存不足时明确提示"XX 零件仅剩 X 件可用"
- 前端带单出库页面 `OutboundScan.vue`

**不带单出库**
- 直接扫描库存看板 → 选数量 → 直接扣减库存
- 批量不带单出库
- 前端不带单出库页面 `DirectOutboundScan.vue`

**转包单管理**
- RepackOrder、RepackOrderItem、RepackKanban 实体与 Repository
- 转包单新建接口（选供应商 → 批量添加零件 → 校验库存 → 保存）
- 转包单删除、搜索接口

**转包作业（扫描执行）**
- 通用转包扫描接口：扫描未入库/已入库/未出库看板 → 生成转包单和转包看板
- 转包看板扫描后选择仓库/库位/器具 → 扣减原库存 + 增加目标库存
- 转包库存流水双向记录
- 前端转包作业页面 `RepackScan.vue`、转包结余 `RepackBalance.vue`

**测试文档**
- 测试计划、功能测试用例、异常测试用例、测试结果记录

**扫码入库 `KanbanScan.vue`**
- 输入看板号或摄像头扫码 → 显示详情 → 确认入库
